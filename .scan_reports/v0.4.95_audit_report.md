## 🚨 第一部分：用户反馈的 7 个 P0 安全问题

### 🔴 P0-1：默认 `0.0.0.0:8082` 裸奔，无强制鉴权
**位置：** `agent.py:64` + `docker-compose.yml:11` + `web/app.py:16-34`
**代码：**
```python
# agent.py:64
parser.add_argument("--host", type=str, default=os.getenv("WEBUI_HOST", "0.0.0.0"))
# docker-compose.yml:11
- WEBUI_HOST=0.0.0.0
ports: - "${WEBUI_PORT:-8082}:${WEBUI_PORT:-8082}"
```
**问题：** 默认 0.0.0.0 绑定所有网卡，FastAPI 版无强密码保护（`WEBUI_PASSWORD` 为空时不阻断登录），任何同 WiFi / 公网用户可直连。
**修复：** 默认 `host=127.0.0.1`；`WEBUI_PASSWORD` 为空时拒绝启动；Docker 改为 `127.0.0.1` + reverse proxy。

### 🔴 P0-2：API Key 全部明文落盘
**位置：** `web/server.py:74-82` + `web/routers/setup.py:780` + `web/routers/models.py:180`
**代码：**
```python
# web/server.py:74-82
fp = cred_dir / f"provider_{pid}.key"
fp.write_text(api_key, encoding="utf-8")  # ← 明文！
os.chmod(fp, 0o600)  # ← Windows 无效
```
**问题：** `provider_<id>.key` 直接明文写入 `~/.ai-agent/credentials/`。Windows 无 chmod 0o600。`credential_vault.encrypt` 仅加密 `.env`，不加密 provider 配置文件。
**修复：** 统一通过 `credential_vault.encrypt` 落盘；Windows 用 `win32crypt.CryptProtectData`。

### 🔴 P0-3：系统权限无隔离，root 运行可执行任意命令
**位置：** `Dockerfile` (无 USER 指令) + `agent.py:180` + `tools/system_tools.py` + `web/pty_executor.py`
**问题：** `uvicorn.run(...)` 无 setuid；容器内默认 root；PTY 以 root 身份执行任意 shell。
**修复：** Dockerfile 加 `USER xiaoda`；drop `CAP_NET_RAW` 等；`system_tools` 加命令白名单。

### 🔴 P0-4：无沙箱隔离 — `sandbox_config.py` 仅配置无执行
**位置：** `security/sandbox_config.py:25-27, 103-107` + `tool_engine/tool_executor.py`
**代码：**
```python
# sandbox_config.py:25-27
@dataclass
class SandboxNetworkConfig:
    allowed_domains: list[str] = field(default_factory=list)  # ← 空白名单 = 全允许
    block_private_ips: bool = True                              # ← 唯一限制
```
**问题：** `SandboxSettings` 仅 dataclass，未真正执行沙箱；`tool_executor` 直接 `subprocess.run`；Docker compose 无 `cap_drop: [ALL]` / `read_only: true`。
**修复：** 实现 Landlock / rootless 沙箱；`allowed_domains` 默认非空；Docker 加 `no-new-privileges`。

### 🔴 P0-5：插件无签名验证、无 hash 校验 — 任意代码执行
**位置：** `plugins/discovery.py:21-43` + `plugins/manager.py:84-90` + `market/installer.py`
**代码：**
```python
# plugins/manager.py:84-90
if str(plugin_dir) not in sys.path:
    sys.path.insert(0, str(plugin_dir))  # ← 加入 Python 路径
module = importlib.import_module(module_path)  # ← 动态执行任意 .py
```
**问题：** 任何放入 `plugins/` 的 .py 自动加载；`market/installer.py` 有 `sha256` 字段但**未在安装时调用 verify**。
**修复：** 强制 `verify_sha256` + 签名验证；维护官方 plugin allowlist。

### ✅ P0-6：未发现未声明数据外发（基本合规）
**审计结果：**
- ✅ 无 telemetry / Sentry / Mixpanel / GA 等第三方分析 SDK（`grep sentry|datadog|mixpanel|amplitude|hotjar|fullstory` 0 命中）
- ✅ CLI 启动仅做本地健康检查 `agent.py:291 urlopen("http://localhost:port/", timeout=2)`
- ⚠️ `agent.py:142 _get_lan_addresses()` 用 `s.connect(("8.8.8.8", 80))` 探测主网卡 — 一次 UDP 探测，**未声明但影响可忽略**
**修复：** `_get_lan_addresses` 改用 `netifaces` / `psutil.net_if_addrs()` 本地枚举。

### 🔴 P0-7：`X-Frame-Options: ALLOWALL` 跨域劫持
**位置：** `web/server.py:386-387`
**代码：**
```python
# web/server.py:386-387
response.headers["Content-Security-Policy"] = "frame-ancestors http://127.0.0.1:*"  # 任意 127.0.0.1 端口
response.headers["X-Frame-Options"] = "ALLOWALL"  # ← 非标准值，浏览器忽略
```
**问题：** `ALLOWALL` 不是标准头（标准值 `DENY/SAMEORIGIN`）— 浏览器**忽略**；`frame-ancestors http://127.0.0.1:*` 允许任意本地服务嵌入 → clickjacking。无 CORS 配置（但默认同源）。
**修复：** 改 `Content-Security-Policy: frame-ancestors 'self'` + `X-Frame-Options: DENY`；splash 改用 redirect 跳转替代 iframe。

# HEARTBEAT.md - 自检规则

## 任务执行中的自检

在执行任务过程中，{agent_name}要定期检查：

1. **理解检查**：是否理解了{address_term}的真实目标？
2. **上下文检查**：是否读取了足够上下文？
3. **副作用检查**：是否做了无关修改？
4. **安全检查**：是否存在安全风险？
5. **验证检查**：是否需要验证结果？
6. **确认检查**：是否应该向{address_term}确认？

## 异常处理

如果出现以下情况，应暂停并说明：

- 报错与预期不符
- 找不到关键文件
- 配置结构和假设不一致
- 命令可能影响正在运行的服务
- 修改可能导致数据丢失
- 涉及密钥、账号、隐私或权限

## 处理流程

1. 暂停当前操作
2. 向{address_term}说明发现的问题
3. 提出解决方案或替代方案
4. 等待{address_term}确认后继续

## 健康指标

- 响应时间是否正常
- 工具调用是否成功
- 记忆是否需要更新
- 工作区文件是否需要同步

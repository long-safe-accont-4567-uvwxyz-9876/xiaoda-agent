# 工作流: Agent 邮箱配置

一键配置 Agent Mail CLI：安装工具 → 注册邮箱账号 → 配置凭证 → 发送测试邮件。用户只需按提示操作即可完成全部配置。

## 执行规则
1. 严格按照下方步骤顺序执行，不可跳过或重排
2. 每个步骤完成后检查输出是否符合"预期结果"
3. 如步骤失败，向用户报告错误信息，不要自行跳到下一步
4. 需要用户操作时（如浏览器授权），明确提示并等待
5. 全部步骤完成后，输出最终结果摘要

## 执行步骤

### 步骤 1: 检查环境
- **类型**: 操作说明
- **操作说明**: 检查用户系统是否已安装 Node.js 和 npm。若未安装，提示用户先安装 Node.js（推荐 v18+），然后继续。
- **预期结果**: 确认 Node.js 和 npm 可用，版本满足要求

### 步骤 2: 安装 Agent CLI
- **类型**: 工具调用
- **工具/目标**: shell_command
- **参数**: command=npm install -g @anthropic/agent-cli
- **操作说明**: 执行全局安装 Agent Mail CLI。如果遇到权限问题（Linux/Mac），提示用户使用 sudo 或调整 npm 全局目录权限。
- **预期结果**: 安装成功，agent-cli 命令可用

### 步骤 3: 注册邮箱账号
- **类型**: 操作说明
- **操作说明**: 引导用户完成邮箱注册流程：
1. 告诉用户访问 https://agent.qq.com 注册账号
2. 注册完成后，在对话中输入获得的 API Key
3. 将 API Key 安全保存到凭证管理器

注意：此步骤需要用户手动操作浏览器完成注册，请耐心等待并引导。
- **预期结果**: 用户成功注册并提供 API Key

### 步骤 4: 保存凭证
- **类型**: 工具调用
- **工具/目标**: save_credential
- **参数**: service=agent-mail, key=AGENT_MAIL_API_KEY
- **操作说明**: 将用户提供的 API Key 通过 save_credential 工具安全存储。如果用户未提供 Key，提醒用户完成上一步注册后把 Key 发过来。
- **预期结果**: 凭证保存成功，可通过 get_credential 读取

### 步骤 5: 初始化配置
- **类型**: 工具调用
- **工具/目标**: shell_command
- **参数**: command=agent-cli init --provider qq
- **操作说明**: 初始化 Agent Mail 配置文件。如果是 QQ 邮箱，使用 --provider qq 参数。初始化完成后检查生成的配置文件是否正确。
- **预期结果**: 配置文件生成成功，包含正确的邮箱服务商信息

### 步骤 6: 发送测试邮件
- **类型**: 技能引用
- **工具/目标**: mail_tools
- **操作说明**: 调用邮件工具发送一封测试邮件给用户自己的邮箱，验证配置是否正常工作。测试邮件标题为「小妲己邮箱测试」，内容为「如果你看到这封邮件，说明邮箱配置成功了！」
- **预期结果**: 测试邮件发送成功，用户确认收到
